# Privacy и данные

## Минимальный состав данных

Платформа хранит только данные, необходимые для обучения:

- имя или учебный псевдоним;
- email для входа;
- роль;
- принадлежность к группе;
- submissions, reports, reviews, scores;
- audit events.

Для MVP не требуются паспортные данные, телефон, адрес, платежные данные или реальные данные
третьих лиц.

## Evidence и attachments

Evidence может содержать:

- скриншоты lab;
- HTTP request/response без cookies и tokens;
- фрагменты уязвимого кода;
- краткое описание payload.

Запрещено прикладывать:

- реальные персональные данные;
- рабочие secrets;
- токены доступа;
- данные внешних систем;
- payload, направленный на ресурсы вне платформы.

## Logs и audit

Logs должны хранить только техническую информацию:

- id пользователя или service identity;
- id объекта;
- статус операции;
- sanitized error;
- timestamp.

Audit events должны быть неизменяемыми для обычных пользователей.

## Retention

| Тип данных | MVP retention |
| --- | --- |
| Reports | До конца защиты и handoff |
| Attachments/evidence | До конца защиты и handoff |
| Technical logs | 30 days |
| Audit events | 180 days |
| Demo accounts | До reset demo data |

Production-ready версия должна иметь configurable retention и процедуру удаления данных.

## Что объяснить в ВКР

- Платформа минимизирует состав персональных данных.
- Практические задания выполняются на synthetic lab data.
- Evidence очищается от secrets и реальных ПДн.
- Контейнеры студентов изолируются от платформенных данных.
- Audit фиксирует действия без раскрытия содержимого приватных отчетов.
