# Content versioning and licensing

## Версии

Версионируются:

- course;
- module;
- lesson;
- assignment;
- rubric;
- payload examples.

Формат версии: `major.minor.patch`.

## Изменение опубликованного материала

- Minor правки текста можно публиковать без пересдачи.
- Изменения критериев оценки требуют новой версии rubric.
- Изменение задания после старта модуля фиксируется в changelog.
- Опубликованные отчеты оцениваются по версии rubric, действовавшей на момент submission.

## Атрибуция

Материалы OWASP Top 10 должны ссылаться на OWASP как источник классификации и терминологии.
Внешние статьи, payload collections и code snippets требуют ссылки на источник и license note.

## Примеры кода и payloads

- Примеры кода используются только в учебных целях.
- Payloads должны быть ограничены lab scope.
- Запрещено включать payloads, нацеленные на реальные внешние системы.

## Changelog

Каждое значимое изменение контента фиксирует:

- дату;
- автора;
- объект;
- старую и новую версию;
- причину изменения.
